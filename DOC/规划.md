# 股票数据获取与选股系统规划

## 1. 系统概述

本系统是一个集股票数据获取、技术指标计算和智能选股于一体的自动化系统。主要功能包括：
- 获取最近150个交易日的股票数据
- 计算技术指标（89日移动平均线等）
- 实现智能选股策略
- 支持增量更新和定时任务

## 2. 技术栈

- **编程语言**: Python 3.8+
- **数据库**: SQLite
- **数据源**: 
  - Tushare (主要数据源)
  - 腾讯财经 (备用数据源)
  - 新浪财经 (备用数据源)
- **技术指标库**: TA-Lib 或 pandas 计算

## 3. 核心功能

### 3.1 数据获取模块
- 从Tushare获取股票日线数据
- 获取最近150个交易日的K线数据
- 支持多数据源切换和容错
- 增量更新机制（全量新增 + 增量补充）

### 3.2 数据存储模块
- 使用SQLite数据库本地存储
- 设计合理的数据表结构
- 支持技术指标存储
- 数据完整性校验

### 3.3 技术指标计算模块
- 89日移动平均线计算
- 成交量分析（放量判断）
- 涨跌幅计算
- 其他技术指标扩展支持

### 3.4 智能选股模块
- 午盘选股策略（11:30收盘时）
- 放量上涨判断

- 涨幅限制筛选（<8%）

## 4. 数据库设计

### 4.1 股票基本信息表 (stock_info)
```sql
CREATE TABLE stock_info (
    ts_code TEXT PRIMARY KEY,    -- Tushare股票代码
    symbol TEXT,                  -- 股票代码
    name TEXT,                    -- 股票名称
    area TEXT,                    -- 所在地域
    industry TEXT,                -- 所属行业
    list_date TEXT,               -- 上市日期
    updated_at TEXT               -- 更新时间
);
```

### 4.2 日线数据表 (daily_data)
```sql
CREATE TABLE daily_data (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    ts_code TEXT,                 -- 股票代码
    trade_date TEXT,              -- 交易日期
    open REAL,                    -- 开盘价
    high REAL,                    -- 最高价
    low REAL,                     -- 最低价
    close REAL,                   -- 收盘价
    pre_close REAL,               -- 昨收价
    change REAL,                  -- 涨跌额
    pct_chg REAL,                 -- 涨跌幅
    vol REAL,                     -- 成交量(手)
    amount REAL,                  -- 成交额(千元)
    ma89 REAL,                    -- 89日移动平均线
    volume_ratio REAL,            -- 量比（成交量/前5日平均）
    is_breakthrough INTEGER,      -- 是否突破89日线（0:否, 1:是）
    UNIQUE(ts_code, trade_date)
);
```

### 4.3 技术指标表 (technical_indicators)
```sql
CREATE TABLE technical_indicators (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    ts_code TEXT,                 -- 股票代码
    trade_date TEXT,              -- 交易日期
    ma5 REAL,                     -- 5日移动平均线
    ma10 REAL,                    -- 10日移动平均线
    ma20 REAL,                    -- 20日移动平均线
    ma30 REAL,                    -- 30日移动平均线
    ma60 REAL,                    -- 60日移动平均线
    ma89 REAL,                    -- 89日移动平均线
    ma120 REAL,                   -- 120日移动平均线
    ma250 REAL,                   -- 250日移动平均线
    volume_avg_5 REAL,            -- 5日平均成交量
    volume_avg_10 REAL,           -- 10日平均成交量
    volume_ratio REAL,            -- 量比
    rsi_6 REAL,                   -- 6日RSI
    rsi_12 REAL,                  -- 12日RSI
    rsi_24 REAL,                  -- 24日RSI
    updated_at TEXT,              -- 更新时间
    UNIQUE(ts_code, trade_date)
);
```

### 4.4 选股结果表 (stock_selection)
```sql
CREATE TABLE stock_selection (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    ts_code TEXT,                 -- 股票代码
    trade_date TEXT,              -- 选股日期
    selection_time TEXT,          -- 选股时间（11:30）
    open_price REAL,              -- 开盘价
    current_price REAL,           -- 当前价（11:30收盘价）
    ma89_value REAL,              -- 89日线值
    volume_ratio REAL,            -- 量比
    pct_chg REAL,                 -- 涨幅
    breakthrough_type INTEGER,    -- 突破类型（1:向上突破, 2:向下突破）
    selection_reason TEXT,        -- 选股理由
    created_at TEXT               -- 创建时间
);
```

## 6. 数据获取流程

### 6.1 增量数据更新流程
```
1. 初始化数据库连接和表结构
2. 获取股票列表（A股市场）
3. 计算最近150个交易日的日期范围
4. 遍历股票列表进行增量更新：
   a. 查询本地已有数据的最新日期
   b. 如果无数据：获取150个交易日全量数据
   c. 如果有数据：获取缺失的交易日数据
   d. 数据清洗和格式验证
   e. 批量插入到数据库
   f. 记录更新日志
5. 计算技术指标（89日线等）
6. 输出更新统计信息
```

### 6.2 技术指标计算流程
```
1. 获取需要计算技术指标的股票列表
2. 对每只股票计算：
   a. 89日移动平均线
   b. 量比（当前成交量/前5日平均成交量）
   c. 判断是否突破89日线
   d. 保存技术指标到数据库
3. 批量更新技术指标表
```

## 7. 选股策略逻辑

### 7.1 午盘选股条件（11:30收盘时）
1. **放量上涨条件**：
   - 量比 > 1.5（成交量放大50%以上）
   - 当前涨幅 > 0（股价上涨）

2. **89日线穿越条件**：
   - 最低价 < 89日线（开盘或盘中曾低于89日线）
   - 当前价（11:30收盘价） > 89日线
   - 形成向上突破形态

3. **涨幅限制条件**：
   - 涨幅 < 8%（避免追高风险）

### 7.2 选股算法流程
```
1. 获取当前日期和时间（11:30）
2. 查询所有A股股票的实时数据（11:30收盘价）
3. 计算每只股票的技术指标：
   a. 当前涨幅 = (当前价 - 昨收价) / 昨收价 * 100%
   b. 量比 = 当前成交量 / 前5日平均成交量
   c. 89日线值（从技术指标表获取）
4. 筛选符合条件的股票：
   a. 量比 > 1.5
   b. 涨幅 > 0 且 < 8%
   c. 最低价 < 89日线 且 当前价 > 89日线
5. 保存选股结果到数据库
6. 输出选股结果列表
```
